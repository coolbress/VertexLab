# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it via the GitHub Security tab:
- Open a private security advisory in GitHub (preferred and monitored).

Please include:
- A description of the vulnerability and affected components
- Steps to reproduce, if possible
- Any known mitigations

We will acknowledge receipt within 5 business days and provide a timeline for a fix when possible.

## Disclosure Policy
- We practice coordinated disclosure. We aim to release a fix before publicly disclosing details.
- If a workaround exists, we may share it privately with reporters before a full fix is released.
- We will credit reporters (if desired) after a patch is available and users have reasonable time to update.

## Supported Versions
- Active development is on the default branch. Security fixes are applied to the latest minor series.
- Older versions may not receive fixes. Please upgrade to the latest release for security updates.
